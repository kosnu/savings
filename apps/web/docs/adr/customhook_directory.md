# ADR: カスタムフックを配置するディレクトリについて

## ステータス

承認済み

## コンテキスト

hooks/ や utils/ 、 libs/ といったディレクトリにカスタムフックを配置する方法が考えられます。適切なディレクトリに配置することで、コードの可読性、保守性、拡張性が向上します。

## 決定事項

utils/ だけに配置し、 hooks/ や libs/ ディレクトリは作成しないことにします。

## 理由

1. **シンプルさ**: カスタムフックを utils/ にまとめることで、ディレクトリ構成がシンプルになり、開発者がコードベースを理解しやすくなります。
2. **一貫性**: すべてのユーティリティ関数とカスタムフックが同じディレクトリにあるため、一貫性が保たれます。
3. **保守性**: カスタムフックが utils/ にあることで、関連するユーティリティ関数と一緒に管理でき、保守が容易になります。
4. **拡張性**: 将来的に新しいユーティリティ関数やカスタムフックを追加する際に、既存のディレクトリ構成に従うことで、一貫性を保ちながら拡張できます。

## 影響範囲

このディレクトリ構成の変更は、プロジェクト全体に影響を与えます。既存のコードベースを新しい構成に移行する際には、インポートパスの更新や関連するドキュメントの修正が必要となります。開発チーム全体でこの変更を理解し、遵守することが重要です。
