# ADR: ディレクトリ構成について

## ステータス

承認済み

## コンテキスト

プロジェクトのディレクトリ構成は、コードの可読性、保守性、拡張性に大きな影響を与えます。適切なディレクトリ構成を採用することで、開発者がコードベースを理解しやすくなり、新しい機能の追加やバグ修正が容易になります。

## 決定事項

以下のディレクトリ構成を採用します：

```
features/<feature-name>/
  components/       # その機能に特化したUIコンポーネント
  hooks/            # その機能に特化したカスタムフック
  utils/            # その機能に特化したユーティリティ関数
  types/            # その機能に特化した型定義
  <feature-name>.tsx # 機能のエントリーポイントコンポーネント
```

この構成により、各機能が独立して管理され、関連するコードが一箇所にまとまるため、開発者が特定の機能に関連するコードを簡単に見つけられます。

## 理由

1. **モジュール性の向上**: 各機能が独立したディレクトリにまとめられることで、モジュール性が向上し、コードの再利用が促進されます。
2. **可読性の向上**: 関連するコードが一箇所にまとまるため、開発者がコードベースを理解しやすくなります。
3. **保守性の向上**: 機能ごとにコードが整理されているため、バグ修正や機能追加が容易になります。
4. **拡張性の向上**: 新しい機能を追加する際に、既存のディレクトリ構成に従うことで、一貫性を保ちながら拡張できます。

## 影響範囲

このディレクトリ構成の変更は、プロジェクト全体に影響を与えます。既存のコードベースを新しい構成に移行する際には、インポートパスの更新や関連するドキュメントの修正が必要となります。開発チーム全体でこの変更を理解し、遵守することが重要です。
