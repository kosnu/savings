# Monthly Budget Domain Rules

## Rules

- 月次予算はBookに属する。
- 月次予算は対象月の月初日を有効開始日として保存する。
- 月次予算の対象年と対象月は有効開始日から決まる。
- 同一Book・同一年・同月の月次予算は重複できない。
- 月次予算レコードは金額あり予算または予算なし状態を表す。
- 金額あり予算は `amount` を持ち、予算なし状態は `amount` を持たない。
- 作成時に所有先Bookが未指定の場合、認証ユーザーのdefault bookを使う。
- 月次予算の所有先Bookは更新で変更できない。
- 対象月の有効状態は、対象月末日以前に開始したレコードのうち最新の1件で決まる。
- 最新の有効状態が予算なしの場合、さらに過去の金額あり予算は対象月に復活しない。
- 月次予算の書き込みでは、Webなどの呼び出し元からdate-onlyの対象月と、対象月範囲判定に使うdate-onlyの基準月を明示して渡す。
- 月次予算の更新、削除または無効化で使うdate-only基準月は、画面レンダー時点で保持した値ではなく、write実行直前に算出した値にする。
- 当月に有効な過去月開始の金額あり予算を更新または削除する場合、過去月の表示を変えずに当月開始の新しい状態として保存する。
- 最新の月次予算を削除する操作は物理削除ではなく、当月以降を予算なしにする状態変更として扱う。
- 月次予算の作成、更新、削除または無効化の対象月は、Web由来のdate-only基準月で見た現在月以降に限る。
- 月次予算の現在月以降判定では、対象月と許可範囲の基準月を同じdate-only基準に揃え、staleなWeb由来基準月やDBサーバー時刻で基準月を補完しない。
- 月次予算の更新、削除または無効化は、過去月の表示結果を変更してはいけない。
- 月次予算の金額は金額ドメインのルールに従う。
- 月次支出合計は、対象月の支払い合計として扱う。
- 予算使用状況は金額あり予算が有効な場合だけ、月次予算額から月次支出合計を差し引いて計算する。
- 予算なし状態または未設定状態では、予算使用状況の差分は表示しない。
- 差分が0以上なら残額、負なら超過額として扱う。
