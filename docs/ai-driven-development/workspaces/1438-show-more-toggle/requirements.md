# Requirements / PRD: Show more の表示切り替え体験を改善する

## 背景と課題

支払い一覧のカテゴリ別サマリーは、初期表示を抑えつつ、必要なときに `Show more` で残りのカテゴリ別合計を確認できる。これは一覧全体の表示領域を守りながら、追加情報へアクセスできるようにするための操作である。

現在の `Show more` は、追加表示の操作として認識しづらく、限られた表示領域やタッチ操作では扱いづらい。また、視覚的な見え方によってはバッジなどの状態表示要素と見間違いやすく、操作要素として理解しづらい。一度展開すると元の表示量へ戻す操作もないため、ユーザーが確認を終えた後もカテゴリ別サマリーが表示領域を取り続ける。

この要求は GitHub Issue #1438 を入力として整理する。Issue は最終仕様ではなく、Requirements / PRD のための意図、境界、成功条件として扱う。

## 対象ユーザーと利用シーン

- 対象ユーザー: savings の Web UI で支払い一覧とカテゴリ別サマリーを確認する認証済みユーザー。
- 主な利用シーン: 月ごとの支払い一覧を確認しながら、初期表示に含まれないカテゴリ別合計を必要に応じて追加確認する。
- 限られた表示領域での利用シーン: スマートフォンなどの狭い画面で、カテゴリ別サマリーを展開して確認したあと、支払い一覧の表示領域を取り戻したい。
- 関連する現行体験:
  - カテゴリ別サマリーは初期表示で最大 3 件に抑えられる。
  - 初期表示に含まれないカテゴリ別サマリーがある場合、`Show more` で取得済みデータの残りを表示できる。
  - `Show more` 後は残りのカテゴリ別サマリーが表示され、追加取得は行わない。
  - 展開後に元の表示量へ戻す操作はない。

## ユーザーストーリー

- 認証済みユーザーとして、初期表示に含まれないカテゴリ別合計を見たいときに、追加表示操作をすぐ認識して押せるようにしたい。そうすることで、必要なカテゴリ別サマリーへ迷わずアクセスできる。
- 認証済みユーザーとして、`Show more` をバッジなどの状態表示ではなく操作要素として理解したい。そうすることで、追加表示できることを見落とさず、誤認せずに操作できる。
- 認証済みユーザーとして、カテゴリ別サマリーを展開した後に元の表示量へ戻したい。そうすることで、確認を終えた後に支払い一覧の表示領域を取り戻せる。
- 認証済みユーザーとして、現在の表示状態と次に起きる操作を直感的に理解したい。そうすることで、展開できるのか、折りたためるのかを迷わず判断できる。

## スコープ

### 対象

- カテゴリ別サマリーの追加表示操作。
- カテゴリ別サマリーの展開後に元の表示量へ戻す操作。
- 追加表示 / 折りたたみ操作の認識しやすさ、押しやすさ、操作ラベル、視覚的優先度を含む UI 体験。
- 展開状態と折りたたみ状態で、ユーザーが現在の状態と次の操作を理解できること。

### 対象外

- カテゴリ別サマリーに表示するデータ内容の変更。
- カテゴリ別サマリーの並び順変更。
- 初期表示件数そのものの変更。
- カテゴリ設定、ピン設定、予算設定の仕様変更。
- サーバーからカテゴリ別サマリーを追加取得するページング。
- DB、API、認証、権限の変更。
- Requirements / PRD 作成 Goal 内での実装、Design Doc 作成、PR 作成。

## 機能要件

### FR-1: 追加表示操作を認識しやすく、押しやすくする

- 初期表示に含まれないカテゴリ別サマリーがある場合、ユーザーは追加表示操作を一覧の追加表示として認識できる。
- 追加表示操作は、限られた表示領域やタッチ操作でも押しやすい。
- 追加表示操作は、周辺のカテゴリ別サマリーとの関係が分かる位置と見た目で表示される。
- 追加表示操作は、視覚的優先度を上げすぎず、バッジなどの状態表示要素と見間違わない見た目で表示される。
- 具体的な配置、サイズ、見た目はこの Requirements / PRD では固定しない。Design Doc で既存 UI の文脈に合わせて決める。

### FR-2: 展開後に元の表示量へ戻せる

- ユーザーが追加表示操作でカテゴリ別サマリーを展開した後、元の初期表示量へ戻す操作ができる。
- 折りたたみ操作は、展開後のカテゴリ別サマリー表示領域から見つけやすい。
- 折りたたみ後は、カテゴリ別サマリーの表示量が初期表示件数に戻る。
- 折りたたみ操作は、取得済みデータやカテゴリ別サマリーの並び順を変えない。

### FR-3: 表示状態と操作結果が分かる

- 折りたたみ状態では、追加表示できるカテゴリ別サマリーがあることが分かる。
- 展開状態では、元の表示量へ戻せることが分かる。
- 操作ラベルやアクセシブルな名前は、現在の表示状態と次に起きる操作をユーザーが理解できる内容にする。
- 最終的な UI 文言はこの Requirements / PRD では固定しない。Design Doc で既存の英語 UI 文言と合わせて決める。

### FR-4: 既存のカテゴリ別サマリー挙動を維持する

- 初期表示件数は既存どおり最大 3 件とする。
- カテゴリ別サマリーが 3 件以下の場合、追加表示 / 折りたたみ操作は不要である。
- 追加表示は、取得済みカテゴリ別サマリーを画面上で表示する操作として扱い、追加の network request を前提にしない。
- 月変更または集計結果変更によってカテゴリ別サマリーの対象が変わった場合、表示量は初期表示件数へ戻る。
- ピン留め優先順、未分類行の扱い、カテゴリ別サマリーの集計内容は変更しない。

## 非機能要件と制約

- 既存 UI の文脈に合う形で改善する。
- Issue で挙げられた具体例を、最終的な配置や見た目の固定仕様として扱わない。
- `Show more` の視覚的優先度を上げすぎず、バッジなどの状態表示要素と見間違わないようにする。
- スマートフォンなどの狭い画面でも操作しやすいことを重視する。
- キーボード操作、スクリーンリーダー向けの操作名、フォーカス可能性を損なわない。
- 新規依存の追加はこの要求の前提にしない。
- カテゴリ表示件数、カテゴリの並び順、ピン、予算、DB、API、認証、権限の仕様判断をこの要求に混ぜない。

## 受け入れ条件

- AC-1: 初期表示に含まれないカテゴリ別サマリーがある場合、追加表示操作が表示される。
- AC-2: 追加表示操作は、一覧の追加表示として認識しやすく、狭い画面やタッチ操作でも押しやすい。
- AC-3: 追加表示操作は、バッジなどの状態表示要素と見間違わない視覚表現になっている。
- AC-4: 追加表示操作を実行すると、取得済みカテゴリ別サマリーの残りを確認できる。
- AC-5: 展開後、元の初期表示量へ戻す操作が表示される。
- AC-6: 折りたたみ操作を実行すると、カテゴリ別サマリーの表示量が初期表示件数へ戻る。
- AC-7: 展開状態と折りたたみ状態で、操作ラベルまたはアクセシブルな名前が次に起きる操作を表している。
- AC-8: カテゴリ別サマリーが 3 件以下の場合、追加表示 / 折りたたみ操作は表示されない。
- AC-9: 追加表示または折りたたみ操作によって、カテゴリ別サマリーの集計内容、並び順、ピン留め優先順は変わらない。
- AC-10: 追加表示はサーバーからの追加取得を必要としない。
- AC-11: 月変更または集計結果変更が起きた場合、カテゴリ別サマリーの表示量は初期表示件数へ戻る。
- AC-12: キーボード操作とスクリーンリーダー向けの操作名が、追加表示と折りたたみのどちらの状態でも破綻しない。

## 技術的考慮事項

- 現行の `CategoryTotals` は、初期表示件数を 3 件にしている。
- 現行の `CategoryTotals` は、表示対象を `categoryTotals.slice(0, visibleCount)` で制限している。
- 現行の `Show more` は、`visibleCount` を取得済みカテゴリ別サマリーの全件数へ更新し、残りを表示する。
- 現行の `Show more` は、展開後に非表示になり、元の表示量へ戻す操作を提供しない。
- 現行の `Show more` は、追加 network request ではなく、取得済み配列の表示範囲を広げる操作である。
- 現行の `CategoryTotals` は、対象月、カテゴリ別サマリーの key、kind、totalAmount、pinned を含む key により、月変更または集計結果変更時に表示状態を初期化できる。
- 既存の `CategoryTotals` のテストには、`Show more` で残りが表示されること、3 件以下では `Show more` が表示されないこと、ピン留め優先順の先頭 3 件を初期表示することの確認がある。
- Design / Plan Goal では、既存の Radix Themes コンポーネント、アクセシブルな名前、タッチ操作での押しやすさ、視覚的優先度、バッジなどの状態表示要素との見分けやすさ、既存のカテゴリ別サマリー layout との整合を確認する。
